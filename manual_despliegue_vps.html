Guía de Despliegue en Servidor VPS

PhishGuard UTB — Plataforma de Concientización
Universidad Técnica de Babahoyo · 6to Semestre 2026

Este documento detalla paso a paso el proceso de configuración y puesta en marcha en producción de la plataforma web PhishGuard UTB bajo tu dominio propio phishguardutb.online.

Requisitos Previos

  1. Un servidor VPS con sistema operativo Ubuntu 22.04 LTS o superior.
  2. Acceso SSH como usuario administrador (root) o permisos con sudo.
  3. Tu dominio phishguardutb.online apuntado en DNS a la IP pública de tu servidor.

Paso 1: Actualizar el Sistema y Dependencias Básicas

Conéctate a tu VPS por SSH y ejecuta los siguientes comandos para actualizar el sistema e instalar las herramientas necesarias:

sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl wget build-essential

Paso 2: Instalar Node.js (v20) y PM2

Instalamos Node.js desde la fuente oficial y el gestor de procesos PM2 de forma global:

# Descargar e instalar Node.js
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Instalar PM2 globalmente
sudo npm install -g pm2

Paso 3: Instalar y Configurar PostgreSQL

Instalamos el motor de base de datos PostgreSQL, creamos la base de datos y el usuario de la aplicación:

# Instalar PostgreSQL
sudo apt install -y postgresql postgresql-contrib

# Iniciar y habilitar el servicio
sudo systemctl start postgresql
sudo systemctl enable postgresql

# Entrar a la consola de PostgreSQL
sudo -i -u postgres psql

Dentro de la consola interactiva de PostgreSQL (psql), ejecuta las siguientes sentencias SQL:

-- 1. Crear base de datos
CREATE DATABASE phishguard_utb;

-- 2. Crear usuario administrador para la app
CREATE USER phishguard WITH PASSWORD 'PhishGuard*UTB*2026';

-- 3. Asignar privilegios completos
GRANT ALL PRIVILEGES ON DATABASE phishguard_utb TO phishguard;

-- 4. Salir de la consola
\q

Paso 4: Clonar el Proyecto y Configurar Variables de Entorno

Clonamos tu repositorio de GitHub en la ruta común para aplicaciones web (/var/www):

# Moverse a la carpeta www
cd /var/www

# Clonar repositorio
sudo git clone https://github.com/AndyMonteroA/phishguard-utb.git
sudo chown -R $USER:$USER /var/www/phishguard-utb
cd phishguard-utb

1. Configurar Backend .env

cd phishguard-backend
cp .env.example .env
nano .env

Modifica los valores del archivo utilizando el editor:

NODE_ENV=production
PORT=5000
DB_HOST=localhost
DB_PORT=5432
DB_NAME=phishguard_utb
DB_USER=phishguard
DB_PASSWORD=PhishGuard*UTB*2026  # La contraseña de base de datos
JWT_SECRET=UnCodigoSuperLargoYSecreto2026!  # Escribe una clave aleatoria fuerte
JWT_EXPIRATION=24h
FRONTEND_URL=https://phishguardutb.online
GOOGLE_CLIENT_ID=568294630065-oeushgct26pbou93mc7poeh1fdj9ivu0.apps.googleusercontent.com

2. Configurar Frontend .env

cd ../phishguard-frontend
nano .env

Ingresa el siguiente contenido:

VITE_API_URL=https://phishguardutb.online/api
VITE_GOOGLE_CLIENT_ID=568294630065-oeushgct26pbou93mc7poeh1fdj9ivu0.apps.googleusercontent.com

Paso 5: Instalar, Estructurar e Iniciar Backend

Instalamos las dependencias de producción, creamos la estructura de tablas y alimentamos la base de datos con los registros del seeder:

cd ../phishguard-backend

# Instalar dependencias de producción
npm install --production

# Crear tablas en la base de datos automáticamente
npm run setup

# Cargar los datos iniciales (Admin, Módulos, Contenidos, Preguntas, Logros)
npm run seed

# Iniciar el backend en segundo plano con PM2
pm2 start src/app.js --name "phishguard-api"

# Configurar PM2 para inicio automático
pm2 save
pm2 startup

Paso 6: Compilar el Frontend

Generamos los archivos estáticos listos para producción del frontend React:

cd ../phishguard-frontend

# Instalar todas las dependencias
npm install

# Compilar proyecto
npm run build

Paso 7: Configurar Servidor Web Nginx

Instalamos Nginx y creamos la configuración del Servidor Web para servir el Frontend y actuar como proxy inverso para la API:

# Instalar Nginx
sudo apt install -y nginx

# Crear archivo de configuración
sudo nano /etc/nginx/sites-available/phishguard

Pega la siguiente configuración dentro del archivo:

server {
    listen 80;
    server_name phishguardutb.online www.phishguardutb.online;

    # Directorio donde se compilo el frontend React
    location / {
        root /var/www/phishguard-utb/phishguard-frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # Redireccionar peticiones al Backend en el puerto 5000
    location /api {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

Habilitamos el archivo de configuración y reiniciamos Nginx:

# Crear enlace simbólico para activar el sitio
sudo ln -s /etc/nginx/sites-available/phishguard /etc/nginx/sites-enabled/

# Desactivar la configuración por defecto de Nginx
sudo rm /etc/nginx/sites-enabled/default

# Probar la sintaxis de Nginx
sudo nginx -t

# Reiniciar Nginx
sudo systemctl restart nginx

Paso 8: Instalar Certificado SSL (HTTPS) Gratis

Importante: Requisito de Seguridad

La autenticación segura de Google OAuth requiere estrictamente que la conexión se realice mediante protocolo seguro HTTPS.

Utilizaremos Certbot para generar de forma automatizada e instalar un certificado SSL válido de Let's Encrypt:

# Instalar snap y certbot
sudo apt install snapd
sudo snap install core; sudo snap refresh core
sudo snap install --classic certbot

# Vincular comando certbot
sudo ln -s /snap/bin/certbot /usr/bin/certbot

# Generar e instalar el certificado en Nginx
sudo certbot --nginx -d phishguardutb.online -d www.phishguardutb.online

Paso 9: Configurar el Cortafuegos (UFW)

Aseguramos el servidor abriendo únicamente los puertos para tráfico seguro:

sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
